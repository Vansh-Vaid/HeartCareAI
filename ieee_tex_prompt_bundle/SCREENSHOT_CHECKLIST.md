# Screenshot Checklist for the HeartCareAI IEEE Paper

Capture screenshots from the running HeartCareAI app and place them in:

`figures/screenshots/`

## Required App Screenshots

1. `home.png` - home or landing page
2. `login.png` - login or registration page
3. `prediction_form.png` - patient input form
4. `prediction_result.png` - probability/risk result page
5. `prediction_history.png` - prediction history page
6. `report_view.png` - generated report view
7. `eda_page.png` - EDA or chart page
8. `chatbot.png` - chatbot interface
9. `doctor_dashboard.png` - doctor page or patient detail view
10. `admin_dashboard.png` - admin dashboard or prediction monitoring page

## Quality Rules

- Use a clean desktop browser screenshot.
- Use a consistent viewport, preferably 1440x900 or 1366x768.
- Use fictional/anonymized patient data only.
- Do not show passwords, API keys, real names, real contact details, or database paths.
- Keep filenames short and descriptive.
- Use 4 to 6 screenshots in the main paper and put extras in an appendix or ZIP folder.

## Recommended Figures for the Paper

- Prediction form
- Prediction result page
- Model comparison chart
- Confusion matrix or ROC curve
- Doctor/admin workflow
- Chatbot support interface

## IEEE Template Reference Screenshots

If screenshots are extracted from the IEEE conference template PDF, keep them in:

`template_screenshots/`

Use those images only as a formatting reference. Do not include them as research figures in the HeartCareAI paper.
